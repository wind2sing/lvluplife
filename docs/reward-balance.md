# 任务周期与奖励规范

本规范用于原版 538 项挑战、自定义任务和任务链步骤。目标是让奖励反映现实行动本身，而不是任务在列表中的位置、玩家当前等级或一个随意的周期倍率。

## 判断顺序

每项任务按以下顺序独立判断：

1. **完成标准**：一次领取究竟需要完成什么，是否跨多天或包含多个阶段。
2. **重复周期**：多久以后，再完成一次仍然是合理的新行动。
3. **任务难度**：综合时长、准备、技能、体力、心理勇气、费用、出行和他人协调成本。
4. **所需精力**：完成当天通常处于低、普通还是高精力状态。
5. **实际属性**：奖励完成行为真正锻炼的属性，不按分类机械套用。

## 重复周期

| 周期 | 使用条件 |
| --- | --- |
| 每日 | 低摩擦、无需预约或专程消费，并且每日重复确有价值的习惯 |
| 每周 | 需要一定准备、恢复、出行或 1–4 小时投入的稳定活动 |
| 每月 | 需要消费、预约、多人协调、专程出行或占用半天的低频活动 |
| 每年 | 季节性活动、年度维护、长途旅行或每年参与一次仍合理的事件 |
| 终身一次 | 特定作品、证照、毕业、首次重大突破、迁居和明确目的地成就 |

周期只控制冷却，不直接改变单次经验。一个困难任务不会仅因为可以每周重复就被压成低奖励；反过来，一项简单的一次性动作也不会仅因“终身一次”获得史诗奖励。

## 难度档位

| Tier | 名称 | 典型范围 | 属性总预算 |
| --- | --- | --- | --- |
| 1 | 轻松一胜 | 10–30 分钟、常规、低门槛 | 2 |
| 2 | 支线任务 | 30–120 分钟，或有适度准备、出行、技能与不适 | 4 |
| 3 | 进阶挑战 | 半天、多阶段，或有明显体力、技能、协调和费用成本 | 7 |
| 4 | 史诗任务 | 多日项目、重大里程碑、高风险、高费用或显著勇气成本 | 12 |

## 经验矩阵

| 难度 | 低精力 | 普通精力 | 高精力 |
| --- | ---: | ---: | ---: |
| Tier 1 | 35 XP | 45 XP | 60 XP |
| Tier 2 | 75 XP | 95 XP | 120 XP |
| Tier 3 | 160 XP | 210 XP | 270 XP |
| Tier 4 | 350 XP | 450 XP | 600 XP |

玩家等级和重复周期不参与经验计算。同一件现实行动不会因为玩家后来等级更高而凭空变得更值钱。

## 属性分配

- Tier 1 通常集中到一个属性，共 2 点。
- Tier 2 可单属性 4 点，或主属性 3 点、副属性 1 点。
- Tier 3 可单属性 7 点，或主属性 5 点、副属性 2 点。
- Tier 4 可单属性 12 点，或主属性 8 点、副属性 4 点。
- 属性根据实际行为判断。例如，组织公益活动更偏魅力与智慧；长途徒步更偏力量与环境；写程序更偏智慧与才能。

## 数据审计

`data/challenge-balance-audit.json` 保存 538 项任务逐条人工判断后的周期、难度、精力、经验、属性和简短理由。它是挑战奖励的审计源，`npm run balance:apply` 会将审计结果同步到正式挑战数据。

`npm run data:audit` 会验证：

- 538 项任务是否全部有独立判断理由；
- XP 是否严格符合固定矩阵；
- 属性点是否符合各 Tier 的总预算；
- 正式挑战数据是否与审计源一致；
- 经典书单是否仍保持 150 项、等级 1、终身一次。

已完成记录会保存当次奖励快照。以后即使再次调整规范，也不会追溯修改过去已经获得的经验和属性。
