# 路线图

路线图优先保证个人长期使用的数据可靠性，再扩展玩法。

## P0：可靠性与正式开源准备

- [ ] 增加明确的云存档状态：保存中、已同步、失败重试和最后同步时间。
- [ ] 为经验、升级、冷却、行动力、撤销、封印和路由增加自动化测试。
- [ ] 增加完整存档导出 / 导入，支持 JSON 备份与灾难恢复。
- [ ] 为存档结构增加版本号和自动迁移机制。
- [ ] 增加错误边界、API 错误提示和附件上传失败恢复。
- [ ] 选择并添加正式开源许可证；目前仓库尚未包含许可证。

## P1：核心玩法增强

玩法扩展遵循三个原则：奖励真实行动而不是点击次数；允许休息、退回和封印，不制造连续打卡焦虑；新增系统必须服务于个人长期使用，而不是堆叠无意义的货币和数值。

### 1. 自定义任务系统

解决原始 538 项挑战无法覆盖个人工作、家庭和长期目标的问题。

最小可用版本：

- [x] 创建、编辑、复制和封印自定义任务。
- [x] 配置标题、说明、分类、等级、经验、属性奖励和重复周期。
- [x] 设置完成标准与完成记录提示，例如“上传结果照片”或“写下三点复盘”。
- [x] 支持“个人任务”的来源标识，与原版挑战区分。
- [x] 根据等级和属性点上限校验奖励，避免自定义任务破坏成长曲线。
- [x] 根据所需精力、重复周期、等级、分类和主要属性自动计算奖励，不估算任务时长。
- [x] 提供全局开关，可关闭创建、编辑、任务链和项目界面而不删除数据。

后续扩展：

- [ ] 自定义分类、图标和颜色。
- [ ] 从 JSON 导入 / 导出个人任务包。
- [ ] 提供“晨间习惯”“家庭维护”“学习计划”等本地模板包。

### 2. 任务类型与组织方式

当前所有内容都表现为单个任务，后续可区分不同现实目标：

| 类型 | 用途 | 示例 |
| --- | --- | --- |
| 快速任务 | 当下容易开始的小行动 | 整理桌面、出门散步 |
| 习惯任务 | 按日、周、月循环 | 每周运动三次 |
| 里程碑 | 只能完成一次的重要节点 | 通过考试、完成一次演讲 |
| 愿望清单 | 不设期限的长期体验 | 去某个国家、学会潜水 |
| 任务链 | 必须依次完成的多步骤冒险 | 选题 → 学习 → 制作 → 发布 |
| 项目 | 包含多个可并行子任务 | 搬家、装修、制作个人网站 |

计划功能：

- [x] 为个人任务增加所需精力和情境标签，不强制估算完成时长。
- [x] 支持任务链的前置步骤、顺序解锁、步骤进度和阶段奖励。
- [x] 支持项目总进度、并行子任务清单和项目完成总结。
- [x] 总等级与分类探索双门槛：高等级角色进入新分类时仍从入门任务开始。
- [x] 不同任务完成数推进分类阶段，封印任务补充候选，已发现任务永久保留。
- [x] 分类探索采用可解释公式：初始 4 项、每个不同完成 +2 名额、每个封印 +1 名额，并在 1 / 3 / 10 个不同完成时开放 Tier 2 / 3 / 4；完整边界规则见 [玩法与机制](gameplay.md#分类探索的精确规则)。
- [ ] 愿望清单默认不进入每日推荐，避免长期目标占据当前行动区。
- [ ] 允许调整任务链步骤顺序、增删步骤以及归档已结束计划。

### 3. 每日冒险板与每周委托

目标不是强迫每天清空列表，而是从大量挑战中给出少量、容易决策的候选行动。

- [x] 每天生成 3 个候选任务：一个轻松任务、一个成长任务、一个自由选择位。
- [x] 推荐时排除冷却和封印任务，并降低最近重复完成分类的权重。
- [x] 允许无惩罚换一批，任务仍可从详情页直接封印。
- [x] 可选择当天状态：低精力、普通、充沛，并据此调整推荐难度。
- [ ] 增加“稍后再做”和最近频繁跳过任务的降权反馈。
- [ ] 每周生成一个跨 2–5 步的主题委托，例如“恢复生活秩序”或“探索所在城市”。
- [ ] 完成整组委托后给予徽章、称号或外观收藏，不额外膨胀属性数值。

### 4. 智能推荐与“为什么推荐”

推荐应当可理解、可纠正，而不是黑箱随机。

- [x] 综合等级、收藏、最近完成分类、属性短板、当前精力、冷却和封印记录排序。
- [x] 显示推荐理由，例如“补足最近较少提升的环境属性”或“你收藏的任务”。
- [ ] 提供“不适合我”“现在没条件”“已经会了”等反馈，并影响后续推荐。
- [ ] 区分永久封印和暂时搁置；搁置任务可在指定日期后重新出现。
- [ ] 支持按地点、情境和精力筛选，例如“在家、安静环境、低精力”。
- [ ] 解释推荐权重变化，让用户知道某项反馈如何影响后续候选。

### 5. 角色成长、专精与称号

六项属性不应只是累计数值，还需要提供阶段反馈和长期变化。

- [x] 每项属性拥有“初识、见习、熟练、精通、大师”独立阶段和里程碑。
- [ ] 根据长期行为生成角色倾向，例如“城市探索者”“知识收藏家”“生活工匠”。
- [x] 允许选择一个当前专精方向，但不限制完成其他类型任务。
- [x] 专精只影响每日推荐权重，不增加属性或经验数值优势。
- [x] 支持六维雷达图和当前属性阶段对比。
- [ ] 增加六维属性的历史变化时间轴。

称号方向：

- [x] 初次启程：完成第一个任务。
- [x] 七日足迹：在七个不同日期留下完成记录。
- [x] 六边形冒险者：六项属性达到指定阶段。
- [x] 归来仍是冒险者：间隔 30 天后重新完成任务。
- [x] 长线主义者：完成任务链或项目。
- [ ] 根据长期行为组合生成更细分的动态称号。

### 6. 徽章、收藏品与非数值奖励

为了保持“真实成就反馈”，奖励应优先成为纪念物，而不是可无限刷取的金币。

- [x] 完成特殊条件后解锁私人称号、徽章、头像框、营地主题与纪念收藏品。
- [x] 收藏馆支持搜索、分类、解锁状态筛选和多种排序。
- [x] 称号、头像框和营地主题可以装备并跨设备同步。
- [x] 收藏馆支持全局关闭，关闭不会删除解锁进度和装备选择。
- [ ] 为终身成就生成独特纪念卡片，可包含日期、文字和照片。
- [x] 每个自然月生成一张个人冒险封面，展示代表任务、完成次数与经验。
- [ ] 允许把单条完成记录制作成可下载的私人成就卡。
- [x] 收藏品只影响外观，不出售、不抽卡、不引入付费数值。

### 7. 完成记录与复盘

- [ ] 编辑完成记录的文字、日期和附件，而不必先撤销任务。
- [ ] 可为记录添加心情、难度感受和“是否值得再次做”。
- [x] 每周营火复盘：自动汇总近七天完成次数、经验、主要属性和图片记录数。
- [ ] 每月回顾：本月代表成就、最常提升属性、被封印或长期搁置的目标。
- [ ] 从复盘中直接调整推荐偏好、创建后续任务或开启任务链。
- [x] 支持全文搜索任务标题、历史记录文字与附件名称。
- [ ] 提供附件灯箱、上传前压缩、替换附件和批量整理能力。

### 8. 连续记录与休息机制

连续天数应当记录生活节奏，而不是变成惩罚。

- [ ] 区分“活跃日”和“休息日”，主动休息不会显示失败或损失经验。
- [ ] 提供每周目标频率，例如一周活跃 3 天，而不是强制每天完成。
- [ ] 连续记录中断后保留历史最佳，不使用付费冻结或补签道具。
- [ ] 长期未使用后启动“重新启程”流程，只推荐一个很小的回归任务。

### 9. 个人统计与成长地图

- [x] 按 7 天、30 天和一年查看经验、完成次数、属性成长与活跃日期。
- [x] 分类热力图：观察现实行动主要分布在哪些生活领域。
- [x] 完成时段统计，帮助观察现实行动节奏。
- [ ] 循环任务坚持率、任务链完成率、收藏转化率和封印原因统计。
- [ ] 属性、分类和任务类型的历史变化趋势。
- [x] 所有统计只在私人存档中计算，不上传公共分析平台。

### 10. 玩法实施顺序

| 阶段 | 首要交付 | 完成标准 |
| --- | --- | --- |
| P1-A（已完成） | 自定义任务 + 精力 / 情境标签 | 能创建个人任务并正常接取、完成、循环、封印和同步 |
| P1-B（已完成） | 每日冒险板 + 可解释推荐 | 首页稳定提供少量候选，可换一批并解释推荐原因 |
| P1-C（已完成） | 任务链 + 项目 | 多步骤目标可保存进度、领取阶段奖励并生成总结 |
| P1-D（已完成） | 每周复盘 + 历史搜索 | 能从完成记录形成周报，并快速找到过去的文字与图片 |
| P1-E（已完成） | 属性阶段 + 称号 + 收藏品 | 成长反馈更丰富，但不影响基础经验平衡 |
| P1-F（已完成） | 个人统计与月度冒险封面 | 能观察长期趋势并生成可保存的私人纪念物 |

P1-A 至 P1-F 的首个可用版本均已完成。下一阶段优先补强完成记录编辑、暂时搁置、每周目标频率、任务链管理和长期趋势时间轴。

## 暂不优先

- 好友、社区动态、公开排行榜。
- 强制打卡提醒或惩罚机制。
- 多租户账号系统。

这些功能会显著扩大隐私、审核和运维范围；在单人体验和数据可靠性稳定前，不作为近期目标。
